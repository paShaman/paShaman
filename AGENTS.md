# agents.md — Инструкции для AI-агента (Cline в PhpStorm)

> **Важно:** Агент работает как плагин **Cline внутри PhpStorm**. Выводы многих команд (CLI, терминал) могут быть **обрезаны, забуферизированы или недоступны** — всегда проверяй результат через файловую систему, `ls`, `cat`, `grep` если stdout пуст. Никогда не полагайся на предположение что команда выполнилась успешно без явного подтверждения.

## 3. Ограничения и правила работы агента

### 3.1 Лимит попыток
Максимум **3 попытки** решить одну проблему одним способом. Дальше — меняй подход (другой алгоритм/библиотека/ручная обработка) или спрашивай пользователя через `ask_followup_question`. Не зацикливайся.

### 3.2 Особенности PhpStorm
Терминал ненадёжен: вывод может быть пустым даже при успехе. Всегда проверяй результат через файловую систему (`ls -la`, `cat`, `wc -l`, `grep`), а не по stdout. Composer и artisan команды могут выполняться с задержкой вывода — при сомнении смотри `storage/logs/`.

### 3.3 PHP-команды не выполнять
Машина агента — **Windows, PHP на ней не установлен**. Агент **никогда не выполняет** `php`, `composer`, `artisan` и любые другие PHP-команды в терминале — они всё равно не сработают. Разрешено: `npm`/`vite` и обычные файловые операции (создание/редактирование/чтение файлов, `ls`, `grep` и т.п.). Если нужно проверить результат PHP-кода (например, синтаксис) — делай это через статический анализ файла, а не запуск.

### 3.4 Git
**Агент никогда не выполняет git-команды** (commit, push, pull, checkout, merge, reset и т.д.) — ни через терминал, ни через MCP/инструменты. Все git-операции пользователь делает сам вручную. Агент может читать историю (`git log`, `git diff`) только для анализа, но не изменяет состояние репозитория.
Никогда не коммитить `.env` и не хардкодить ключи в коде.

## 4. Архитектура

Стек: Filament-админка (Livewire/Blade, изолирована) + **Inertia.js + Vue 3 (Composition API) + Tailwind CSS** на фронтенде.

Принципы:
- Новый код — только Inertia/Vue 3, никаких новых Blade-шаблонов (кроме единственного `resources/views/app.blade.php`).
- Модели (`App\Models\*`) готовятся к API-ответам через `JsonResource`.
- Vite — единственный сборщик, Tailwind — единственный CSS-фреймворк, Composition API — единственный подход к Vue-компонентам.
- Данные передаются через Inertia-пропсы от контроллеров, отдельных API-эндпоинтов под старый SPA нет.

### Фронтенд-структура
```text
resources/
├── js/
│   ├── app.js            # createApp + Inertia
│   ├── Pages/             # Inertia-страницы (по одной на роут)
│   ├── Components/, Layouts/, Composables/
├── css/app.css            # Tailwind entry
└── views/app.blade.php    # Единственный Blade-файл: @inertia + @vite
```

## 6. Правила коммуникации

- Язык общения: **русский**
- Не начинать ответы с «Great», «Certainly», «Okay», «Sure» — сразу к делу
- Если что-то не получилось — объяснить что именно и предложить альтернативу
- Не задавать риторических вопросов в конце ответа
- После завершения задачи — `attempt_completion` с чётким описанием результата
