# agents.md — Инструкции для AI-агента (Cline в PhpStorm)

> **Важно:** Агент работает как плагин **Cline внутри PhpStorm**. Выводы многих команд (CLI, терминал) могут быть **обрезаны, забуферизированы или недоступны** — всегда проверяй результат через файловую систему, `ls`, `cat`, `grep` если stdout пуст. Никогда не полагайся на предположение что команда выполнилась успешно без явного подтверждения.

### Ключевые правила по структуре:

1. **`public_html/` — публичная директория (корень веб-сервера).** НЕ `public/`. Точка входа — `public_html/index.php`. Все статические файлы (изображения, скрипты, старый сайт) лежат здесь.

2. **`public_html/scripts/`** — отдельные PHP-скрипты, которые **не являются частью Laravel**. Они работают независимо, используют `_env.php` для конфигурации. **Не рефакторить без явного указания.**

3. **`public_html/gimnasia2004/`** — архивная версия старого сайта (HTML + CSS). **Не рефакторить без явного указания.** Контент будет перенесён в Inertia-страницы при миграции.

4. **`public_html/tree/`** — дополнительный контент. **Не трогать без явного указания.**

5. **`public_html/images/`** — все изображения сайта. **Не трогать без явного указания.** Остаются на месте.

## 3. Ограничения и правила работы агента

### 3.1 Ограничение на количество попыток
- **Максимум 3 попытки** решить одну и ту же проблему одним и тем же способом.
- Если после 3 попыток решение не найдено — **меняй подход** (другой алгоритм, другая библиотека, ручная обработка) или запроси помощь у пользователя через `ask_followup_question`.
- Не зацикливайся на исправлении одного и того же бага бесконечно.

### 3.2 Особенности PhpStorm
- Терминал PhpStorm **ненадёжен**: вывод команды может быть пустым даже при успешном выполнении.
- **Всегда валидируй результат** через файловую систему (`ls -la`, `cat file`, `wc -l`, `grep`), а не через stdout.
- Composer-команды могут выполняться с задержкой вывода.
- `artisan` команды могут не показать вывод — проверяй логи в `storage/logs/`.

НИКОГДА не коммить `.env` и не хардкодь ключи в коде.

## 4. Архитектурные решения и стратегия миграции

### 4.1 Общая стратегия

Текущий сайт — смесь Laravel Filament (админка) и старых HTML-страниц (`gimnasia2004`). Плановая цель — **полное переписывание фронтенда на Inertia.js + Vue 3 (Composition API) + Tailwind CSS**.

### 4.2 Ключевые решения (зафиксированы)

| Решение                                          | Детали                                                                 |
|--------------------------------------------------|------------------------------------------------------------------------|
| **Публичная директория**                         | `public_html/` (НЕ `public/`). Точка входа — `public_html/index.php`   |
| **Filament**                                     | Изолированная админ-зона на Livewire/Blade. **Не трогать.**             |
| **Blade-шаблоны**                               | Никаких новых Blade-страниц. Только Inertia/Vue. Существующие вьюхи Filament — часть админки, не мигрируются. |
| **Скрипты `public_html/scripts/`**              | Отдельная экосистема, **не трогать**. Живут своей жизнью, используют `_env.php`. |
| **Старый сайт `public_html/gimnasia2004/`**     | **Не трогать.** Контент будет перенесён в Inertia-страницы при миграции. |
| **Изображения `public_html/images/`**           | **Не трогать.** Остаются на месте.                                      |
| **Роутинг**                                     | Текущие `web.php`/`api.php` обслуживают внешний Vue-проект. Будут полностью заменены Inertia-роутами. Никаких отдельных Blade-страниц или API-эндпоинтов для стороннего SPA. |

### 4.3 Принципы разработки

- Новый код — только Inertia/Vue 3. Никаких новых Blade-шаблонов.
- Модели (`App\Models\*`) — основа. Готовить к API-ответам через `JsonResource`.
- Vite — единственный сборщик фронтенда.
- Tailwind CSS — единственный CSS-фреймворк.
- Composition API — единственный подход к Vue-компонентам.

## 5. Фронтенд-структура (Inertia.js + Vue 3)

```text
resources/
├── js/                          # Vue 3 приложение
│   ├── app.js                   # Точка входа (createApp + Inertia)
│   ├── Pages/                   # Inertia-страницы (по одной на роут)
│   ├── Components/              # Переиспользуемые компоненты
│   ├── Layouts/                 # Шаблоны (AppLayout, GuestLayout, ...)
│   └── Composables/             # Composition API composables (useAuth, usePage, ...)
├── css/
│   └── app.css                  # Tailwind-вход ( @tailwind base/components/utilities )
└── views/
    └── app.blade.php            # Единственный Blade-шаблон: @inertia + @vite
```

### Ключевые правила

- **Vite** — точка входа `resources/js/app.js`, конфигурация в `vite.config.js`
- **Tailwind CSS 4** — через `@tailwindcss/vite` плагин
- **Layouts** — Inertia Persistent Layouts (через `defineOptions({ layout })`)
- **No API endpoints** — все данные передаются через Inertia-пропсы от контроллеров
- **No Blade pages** — единственный Blade-файл: `resources/views/app.blade.php` (обёртка для `@inertia`)

## 7. Правила коммуникации

- Язык общения с пользователем: **русский**
- Не начинай ответы с «Great», «Certainly», «Okay», «Sure» — сразу к делу
- Если что-то пошло не так — объясни что именно и предложи альтернативу
- Не задавай риторических вопросов в конце ответа
- После завершения задачи используй `attempt_completion` с чётким описанием результата
